# 《拾日》 --- 拾起日子，十日一期


 - 把一年切成36个“十天”，这是我刷抖音时候遇到的概念，我觉得很有意思，所以我想试试
 - 感谢AI的发展，我的Python水平还远达不到开发水准

## 发展方向 ---“钩子” 基于36格框架


 ### 方向一：视觉反馈——把格子涂满


- 核心逻辑：每个十天过去，如果你完成了某个小目标，这个格子就“被点亮”。
  - 支持最简单的“打卡”。
  
  - 用户自定义每个十天的一个小目标（比如“读完一本书”“早睡”）。
  
  - 格子颜色从灰到亮，完成度越高越鲜艳。

吸引人的点：年末生成一张“你的年度色卡”，可保存可分享。

### 方向二：时间锚点——给十天起名字
    
普通日历让人感觉时间是匀速流逝的，但这个工具可以反过来：让用户主动定义每个十天的意义。
   - 核心逻辑：每个十天不是“3月11日-3月20日”，而是用户可以自己命名的时段。
     - 每个格子点进去，第一行就是命名输入框。
     
     - 提供一个随机生成的“灵感词库”，不知道叫什么就摇一个。
     
     - 吸引人的点：年底生成一份“你的年度命名报告”——这一年你给了自己多少个冲刺期、多少个休息期。

这个功能让工具从“记录时间”变成“定义时间”，情绪价值很高。

### 方向三：预支与借贷——时间是可以挪用的
  
  - 核心逻辑：这十天没状态，可以从未来的某十天借一点“高效额度”过来。

    - 每个格子默认有10个“精力点”（或者就叫天数）。

    - 这十天你特别忙，实际只完成了6天的任务，你可以选择从一个月后的那个格子“借”2天过来。

    - 被借的格子会显示“负债状态”，提醒你以后要还。

## 拾日 · 代码架构 v1.0

    shiri/
    ├── index.html          # 入口：36格主视图
    ├── style.css           # 所有样式（初期只有一个文件）
    ├── script.js           # 主逻辑：格子渲染、状态切换、数据读写
    ├── storage.js          # 所有 localStorage 操作（分离出来，以后换后端不痛）
    ├── share.js            # 分享功能：生成色卡/截图
    └── data/               # 预设数据文件夹
        └── presets.js      # 灵感词库、默认配色等

    核心原则：没有用框架，没有分太多文件。现在需要的是跑起来，不是完美架构。
